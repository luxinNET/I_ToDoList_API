# 开发计划 OpenSpec

## 阶段 0：项目初始化

目标：建立可协作、可部署、可持续演进的项目基础。

交付物：

- Maven + Spring Boot 项目骨架。
- `CLAUDE.md`、`README.md`。
- OpenAPI 文档骨架。
- OpenSpec 中文文档。
- GitHub 公开仓库连接。
- 腾讯云部署文档和 Nginx 安全配置示例。

验收标准：

- 项目目录结构清晰。
- 关键技术选型写入文档。
- 不包含任何真实密钥或敏感配置。

## 阶段 1：基础设施

目标：让应用具备基础运行能力和统一工程规范。

开发内容：

- 配置 `dev`、`prod` 环境。
- 配置 PostgreSQL datasource + HikariCP。
- 配置 Flyway 数据库迁移。
- 配置 MyBatis-Plus。
- 配置 Redis + Redisson。
- 配置统一响应结构。
- 配置全局异常处理。
- 配置 OpenAPI / Swagger UI。
- 配置 Actuator / Micrometer。

验收标准：

- 本地环境可以启动应用。
- Flyway 能执行初始化迁移。
- `/actuator/health` 可访问。
- `/v3/api-docs` 在开发环境可访问。

## 阶段 2：认证与用户模块

目标：实现 MVP 登录体系和用户资料能力。

开发内容：

- 用户表、第三方身份表、refresh token 表。
- 邮箱 + 密码注册与登录。
- 手机号 + 密码注册与登录。
- 微信小程序 `code` 登录。
- JWT access token 签发与校验。
- Refresh token rotation。
- 登出当前设备。
- 登出所有设备。
- 当前用户资料查询和修改。
- 登录、注册、刷新接口限流。

验收标准：

- 用户可以通过邮箱或手机号注册登录。
- 微信小程序 code 登录流程具备服务端接口。
- 受保护接口必须携带有效 JWT。
- refresh token 重放会被拒绝。
- 密码和 token 不以明文形式保存或打印。

## 阶段 3：清单与任务 MVP

目标：实现 TodoList 核心业务闭环。

开发内容：

- 默认清单初始化。
- 清单 CRUD。
- 清单排序。
- 任务 CRUD。
- 完成/取消完成任务。
- 标记/取消重要任务。
- 加入/移出“我的一天”。
- 截止日期、提醒时间、备注。
- 任务步骤/子任务 CRUD。
- 任务排序。
- 任务分页与过滤查询。

验收标准：

- 用户可以创建清单并添加任务。
- 用户可以完成、删除、排序任务。
- “我的一天”、“重要”、“计划内”视图可以通过过滤条件构建。
- 用户 A 不能访问用户 B 的清单或任务。

## 阶段 4：标签、提醒与同步

目标：增强多端使用体验。

开发内容：

- 标签 CRUD。
- 任务绑定/解绑标签。
- 提醒记录 CRUD。
- 基础提醒查询。
- `sync_changes` 写入机制。
- bootstrap 同步接口。
- changes 增量同步接口。
- 客户端设备信息记录。

验收标准：

- 客户端可以按标签筛选任务。
- 客户端可以拉取初始化数据。
- 客户端可以按版本拉取增量变更。
- 删除操作可以通过同步接口反映到客户端。

## 阶段 5：部署与安全加固

目标：完成腾讯云轻量服务器生产部署方案。

开发内容：

- systemd service 文件说明。
- Nginx HTTPS 反向代理。
- Nginx 安全响应头。
- 认证接口限流。
- 生产环境 Swagger/Actuator 暴露控制。
- PostgreSQL 备份脚本。
- 健康检查脚本。
- 部署回滚说明。

验收标准：

- 服务器只开放必要端口。
- HTTPS 可用。
- `/actuator/health` 可用于健康检查。
- 数据库和 Redis 不暴露公网。
- 备份脚本可执行并生成备份文件。

## 阶段 6：后续增强

候选能力：

- 共享清单与成员权限。
- 离线写入与冲突合并。
- 推送通知。
- 重复任务执行规则。
- 附件上传。
- 管理后台。
- Prometheus/Grafana 监控面板。

## 推荐开发顺序

1. 先完成认证与用户模块。
2. 再完成清单与任务 MVP。
3. 然后补齐标签、提醒和同步。
4. 最后做腾讯云部署加固。

不要过早实现共享、附件、推送和复杂离线冲突处理，避免 MVP 范围失控。
